# Model Evaluation Record (013)

Card-level accuracy over the curated ES/EN set (`fixtures/cards.ts`, 40 labelled
cards). Scores generated by `npm run bench:sentiment` (2026-07-23); footprints
measured from the downloaded fp32 `model.onnx` files. **This is the decision gate
for feature 013 (FR-001/FR-005/FR-007).**

## Results

| Candidate | Overall | ES | EN | fp32 footprint | vs baseline |
|-----------|---------|----|----|----------------|-------------|
| **Baseline** — distilbert multilingual (previous default) | 72.5% (29/40) | 68.4% | 76.2% | 529 MB (1×) | — |
| twitter-roberta as a single model | 65.0% (26/40) | 31.6% | 95.2% | 481 MB | −7.5 pp |
| 3-model routing (multi + en + es) | **92.5% (37/40)** | 89.5% | 95.2% | 1427 MB (**2.7×**) | +20.0 pp |
| **2-model routing** (es→RoBERTuito, en→twitter-roberta; unknown→en) | **90.0% (36/40)** | 84.2% | 95.2% | 898 MB (**1.7×**) | **+17.5 pp** |

Model ids: multi = `Xenova/distilbert-base-multilingual-cased-sentiments-student`,
en = `Xenova/twitter-roberta-base-sentiment-latest`,
es = `Xenova/robertuito-sentiment-analysis`.

## Decision — ADOPTED: 2-model language-aware routing ✅

Spanish cards → **RoBERTuito**, English cards → **twitter-roberta**, cards whose
language is not confidently detected → the English model as fallback.

| Gate | Requirement | This config | Verdict |
|------|-------------|-------------|---------|
| Card accuracy (SC-001) | ≥ 90% AND ≥ 10 pp over current | 90.0%, +17.5 pp | ✅ |
| Team-mood agreement (SC-002) | mood score matches intended tone on ≥ 90% of boards | 4/4 boards (positive→10, negative→1, neutral→4.6, positive+expected-negativity→7.8) | ✅ |
| Routing margin (FR-007) | ≥ 5 pp over single-multilingual baseline | +17.5 pp | ✅ |
| Download budget (FR-013/SC-006) | ≤ ~2× current footprint | 1.7× (898 MB vs 529 MB) | ✅ |

**Why not the 3-model config** (92.5%): it is only +2.5 pp better but needs a third
model, pushing the footprint to **2.7×** — over the ~2× budget (FR-013). The
`unknown`-language fallback rarely fires on real ES/EN boards (short/empty cards
resolve to neutral before routing), so the 2-model set captures nearly all the gain
at 63% of the download.

**Why the current model was not kept**: at 72.5% it is 17.5 pp below the adopted
config and fails the SC-001 target outright — it is the source of the reported poor
card- and team-level results.

### Notes
- The single English model collapses on Spanish (31.6%), confirming that a
  per-language model — not one general multilingual model — is what lifts accuracy.
- Confidence thresholds (FR-003) were left at the existing permissive values
  (pos/neg 0.4, neu 0.25); the adopted models emit 3-class softmax whose argmax
  drove the 90% above, and the thresholds only gate display/aggregation, which the
  single `isConfident` rule keeps consistent (FR-009). Revisit only if badge
  suppression looks off in the app.
- Footprints are **fp32** (what the worker downloads today, no `dtype` set). Moving
  to quantized (q8) weights would shrink all models ~4× and is a separate, optional
  perf change — it could later make the 3-model config fit the budget.

## Reproduce

```bash
npm run bench:sentiment    # rewrites the results table above (downloads models)
npm run test:accuracy      # asserts the ADOPTED runtime config still hits ≥90% (SC-001)
```
